# AS17.2 实现简报：cohort 选择质量（确定性偏好 profile 兼容的 pan_can_atlas，解锁 promotable）

Status: Assigned to Codex（新分支 feat/cohort-selection-quality，从 main 起，main 已含 AS17.1）
Owner: Claude(编排) · Codex(执行)
Spec source: AS17.1 live 暴露——接地后仍选中 legacy `brca_tcga`(缺 `_rna_seq_v2_mrna` profile)而非 `brca_tcga_pan_can_atlas_2018` → 超时 → rejected
Depends on: AS17.1（grounded cohort inference）

## 背景

AS17.1 把 cohort 接地到真实 study,但 `infer_grounded_cohort_study`(`agent_ops_commands.rs`)的选择逻辑里 **pan_can_atlas 偏好只在"LLM 答案非法"的 `heuristic_best_cbioportal_study` 兜底里生效**。live:LLM 选了**合法但 legacy** 的 `brca_tcga`(在 shortlist 内)→ 直接采纳 → 而 legacy study 缺工具所需的 `_rna_seq_v2_mrna` profile → 运行超时 → 验证门恒 `rejected`,`promotable` 不可达。

AS17.2 让 **profile 兼容的 pan_can_atlas 偏好确定性生效**(即便 LLM 答案合法),使工具能落到带正确 profile 的规范 TCGA 研究上。

## 编排者裁决（约束）

仅改 `crates/agentflow-cli`(AS17.1 的 cohort 选择路径),不改 core/argument.rs/CycleReport/工具库。

### 1. shortlist 不丢 pan_can_atlas（cap 前保护）

`shortlist_cbioportal_studies`/`score_cbioportal_study_match`:对 studyId 含 `pan_can_atlas` 的匹配 study **加分**(排序权重最高),保证它们不被 `GROUNDED_COHORT_SHORTLIST_LIMIT`(20)截断丢弃。

### 2. 选择时确定性偏好 pan_can_atlas 子集

`infer_grounded_cohort_study` 拿到 shortlist 后:
- 计算 `pan_can_subset` = shortlist 中 studyId 含 `pan_can_atlas` 的子集。
- **若 `pan_can_subset` 非空 → 把后续 LLM 选择与启发式兜底都限定在 `pan_can_subset` 内**(LLM prompt 只给 pan_can_subset;LLM 答案必须在 pan_can_subset;否则 `heuristic_best_cbioportal_study(pan_can_subset)`)。
- 若 `pan_can_subset` 为空 → 维持现状(在全 shortlist 上 LLM+heuristic)。
- 这样:某癌种存在 pan_can_atlas 研究时,**一定**落到 pan_can_atlas(profile 兼容);不存在时才退回其它。

### 3. 不变量

- 仅改 CLI;`git diff crates/agentflow-core` 为空;不碰 argument.rs/CycleReport/工具库/DecisionKind。
- 无新依赖;fetch 失败/无匹配/Noop 降级路径不变(AS17.1 行为保持)。
- 无任何具体疾病/study 常量写死;只新增"studyId 含 `pan_can_atlas`"这一通用结构性偏好(已是 AS17.1 既有启发式用语)。

## 测试（离线 stub，扩展 AS17.1 测试）

`agentflow-cli`:
- **核心新增**:shortlist 含 legacy `brca_tcga` 与 `brca_tcga_pan_can_atlas_2018`,**stub LLM 返回 legacy `brca_tcga`** → 最终仍返回 `brca_tcga_pan_can_atlas_2018`(确定性偏好覆盖了合法但次优的 LLM 选择)。
- 多个 pan_can_atlas 候选 + LLM 选其一(合法,在 pan_can_subset)→ 返回该 pan_can_atlas。

- cap 保护:>20 个匹配且 pan_can_atlas 排在原字母序靠后 → pan_can_atlas 仍在 shortlist(未被截断)。
- 既有 AS17.1/AS17 测试保持绿。
