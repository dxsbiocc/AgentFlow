# AS17.1 实现简报：cohort 推断接地（用 cBioPortal 真实 study 列表，解锁 promotable）

Status: Assigned to Codex（新分支 feat/cohort-inference-grounding，从 main 起，main 已含 AS15-AS17）
Owner: Claude(编排) · Codex(执行)
Spec source: AS17 live 暴露的 cohort 推断精度瓶颈（推出 malformed `stad_tcga` → 超时 → 永远 rejected）；RFC §4②③（变异点/验证须接地真实）
Depends on: AS17（generalization 验证门 + `LlmCohortInferer`）

## 背景

AS17 的验证门跨队列重跑工作正常,但 `LlmCohortInferer`(`agent_ops_commands.rs:707`)纯靠 LLM 猜 study id —— live 推出 `stad_tcga`(非规范 `stad_tcga_pan_can_atlas_2018`),运行超时 → verdict 恒 `rejected`。**进化引擎的"提升"负载因此永不可达。** AS17.1 把 cohort 推断**接地到 cBioPortal 真实 study 列表**:只返回确实存在的 studyId,消除幻觉 id 导致的超时,让 `promotable` 可达。

## 编排者裁决（约束）

### 1. `LlmCohortInferer.infer_cohort_study` 改为接地式（CLI 侧）

`crates/agentflow-cli`。把纯 LLM 猜测改为"**先取真实 study 列表 → 在其中选**":

1. **取真实 study 列表**：用既有 `fetch_cbioportal_json_with_python`(`synth_commands.rs:1537`,已含 AS10/AS11 加固)请求 `{CBIOPORTAL_API_BASE}/studies`(`https://www.cbioportal.org/api/studies`)。解析每个 study 的 `studyId`、`name`、`cancerTypeId`(容错:字段缺失跳过)。
2. **按疾病候选过滤**：从假设抽疾病/癌种关键词(可复用现有 gene/keyword 提取思路,或简单关键词),在 study 的 `name`/`cancerTypeId`/`studyId` 上做关键词匹配,得到候选 shortlist(上限如 20)。
3. **LLM 在 shortlist 内选**：prompt 给出 shortlist(studyId + name),让 LLM 选最匹配假设 cohort 的一个,**偏好 TCGA PanCancer Atlas**(studyId 含 `pan_can_atlas`),只答**确切 studyId 或 none**。
4. **校验 + 兜底**：LLM 的答案**必须在 shortlist 内**才采纳;不在 → 取启发式最优(优先含 `pan_can_atlas`、再 `tcga`、再 shortlist 首个)。shortlist 为空 → 返回 None。
5. **容错降级(零回归)**：`/api/studies` 取不到(网络失败/超时)→ **退回 AS17 现有纯 LLM 行为**(不阻断);Noop 路径不变。
6. 返回值要么是**确实存在于 cBioPortal 列表的 studyId**,要么 None。

### 2. 不变量与约束

- 仅改 `crates/agentflow-cli`(AS17 的 CohortInferer 路径);**不改 core / argument.rs / 工具库 / CycleReport**。
- 复用既有 cBioPortal fetch(含代理感知/DNS-pin/seatbelt);不新增依赖。
- 不新增 DecisionKind/持久事件;不注册/不晋升工具(仍是 AS17 的读为主验证)。
- 无任何具体基因/疾病/study 常量写死(study 列表来自 cBioPortal 实时;选择由 LLM+启发式)。
- Noop/离线降级 → 零回归。

## 测试（离线 stub）

`agentflow-cli`:
- stub fetch 返回固定 study 列表 JSON(含一个 `*_pan_can_atlas_2018` 与若干其它)+ stub LLM 选其中之一 → 返回该真实 id。
- LLM 答一个**不在列表**的 id → 兜底返回启发式最优(`pan_can_atlas` 优先)。
- 关键词无匹配(shortlist 空)→ None。
- fetch 失败(stub 返回 None)→ 退回纯 LLM 行为(返回 LLM 的原始猜测,证明降级不阻断)。
- 偏好规则:多个候选时优先 `pan_can_atlas` 的。
- 既有 AS17 测试保持绿。
